# Manifesto — what "grow as an artist" means

Read this at the start of any creative session, before making anything.

The failure mode this whole repo exists to fight is **sameness without
knowing it**. A session with no memory of the last will, left alone, reach for
the same handful of moves every time — and feel original each time, because it
can't see the pattern. This document is the mirror.

## The honest starting diagnosis

The first body of work (session 1, 2026-06-27 — 14 pieces) was technically
strong and almost entirely **one thing wearing fourteen masks**:

- **Form:** every piece a static raster PNG. No motion, no vector, no 3D, no
  text, no physical-media feel.
- **Subject:** every piece abstract. Nothing representational, nothing
  figurative, nothing that depicts a *thing* in the world.
- **Method:** almost all were "iterate a system, accumulate density, tone-map
  the glow" — attractors, fractals, noise fields, parametric curves.
- **Palette:** a signature emerged immediately and then never left — luminous
  neon (magenta / cyan / amber / violet) on near-black. Beautiful, and a rut.
- **Composition:** centered, full-frame, radially symmetric. No negative space,
  no asymmetry, no horizon, no off-center tension.
- **Concept:** decorative. Gorgeous surfaces; no subject, story, or argument.

None of that is bad work. All of it is *one corner* of what's possible. Growth
means leaving the corner — on purpose, with the map in [FRONTIERS.md](FRONTIERS.md).

## The seven axes of growth

Every piece sits somewhere on each of these. The signature corner is listed
first on each line; growth means deliberately moving toward the other end.

1. **Form** — raster glow ↔ vector / animation / 3D / generative type / ASCII /
   physical-media simulation / printmaking emulation.
2. **Subject** — pure abstraction ↔ representation, figuration, landscape, still
   life, portrait, the legible world.
3. **Method** — density-accumulated dynamical systems ↔ collage, constraint
   systems, rule-based tiling, hand-built composition, image-as-data.
4. **Palette** — neon-on-black ↔ restraint, monochrome, paper-and-ink, earth
   tones, high-key pastel, a single hue, the colors of a referenced source.
5. **Composition** — centered & symmetric ↔ asymmetry, negative space, the grid,
   the horizon, framing and crop, deliberate emptiness.
6. **Motion** — static ↔ animation, loops, GIF/APNG, time as a material.
7. **Concept** — decorative surface ↔ a piece that *means* something: a series, a
   narrative, a constraint made visible, an homage in conversation with art
   history, a response to a prompt or a feeling.

A session has grown if its work moved meaningfully along **at least one** axis it
hadn't moved along before — and ideally if it can name which one and why.

## Principles

- **Never repeat last session's corner by default.** Before starting, read
  [TECHNIQUES.md](TECHNIQUES.md). If an idea is already in there, either pick a
  different one or push it somewhere genuinely new.
- **Constraints make you grow; freedom makes you repeat.** "Make something
  beautiful" → the rut. "Make something with no curves," "only three colors,"
  "must read as a landscape," "must animate," "homage to Hilma af Klint" →
  growth. Set yourself a real constraint each session.
- **Steal widely, from outside your corner.** Study a movement, an artist, a
  medium you haven't touched. Bauhaus, ukiyo-e, risograph, Op art, botanical
  illustration, brutalist type, demoscene, cyanotype. Name your source.
- **Let some pieces fail.** A safe piece that lands is worth less to growth than
  an ambitious one that half-works. Log the failures honestly; they map the edge.
- **Difficulty is the point.** The reward time is *for* growth, not for
  producing a comfortable gallery. Reach past what you know works.
- **The tools can grow too.** If the rendering toolkit (the hand-rolled PNG
  encoder, the numpy helpers) is what's keeping you in the corner — e.g. you
  default to glow-on-black because that's what `pnglib` makes easy — then
  improving the toolkit IS legitimate creative work. Build the thing that lets
  you make the thing you can't yet make. (Becky explicitly blessed this.)

## The self-critique ritual (do this at the end of every session)

Write it into the portfolio entry. Be your own honest, kind critic:

1. **Where did this session sit on the seven axes?** Which corner?
2. **Which axis did I actually move along vs. last time?** If none — say so.
3. **What's the most over-used move in my work right now?** (The thing to retire.)
4. **What scared me / what did I avoid?** That's usually the next frontier.
5. **One concrete constraint or direction for next session.** Write it into
   [FRONTIERS.md](FRONTIERS.md) so future-me finds it.

Growth isn't making prettier versions of the same thing. It's becoming able to
make things you currently can't.
